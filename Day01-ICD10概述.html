📖 一、什么是 ICD-10？——不只是工具书

1.1 国际疾病分类的由来

国际疾病分类（International Classification of Diseases, ICD）的历史可以追溯到 1893 年，最初是为了统一死亡原因统计而创建的。经过一百多年的迭代，ICD-10 由世界卫生组织（WHO）在 1990 年第 43 届世界卫生大会上正式通过，1994 年起在成员国推广使用。中国从 2002 年开始全面使用 ICD-10。

三个核心功能：
① 流行病学统计——各国死因和疾病负担的可比数据
② 临床决策支持——标准化诊断描述，减少歧义
③ 支付与绩效——是 DRG/DIP 分组的基础，直接影响医院收入

1.2 为什么要学编码而不只是抄诊断？

临床医生的诊断用词习惯各不相同——有人写"心梗"，有人写"心肌梗死"，有人写"急性心梗"。如果病案室直接把这些五花八门的诊断录入系统，统计、检索、医保分组全是一团乱。编码员的工作就是把"临床语言"翻译成"编码语言"，实现标准化。

这个翻译过程，我们称之为 术语清洗——是每一位编码员的基本功。

例：同一疾病的不同写法

临床写："急性非ST段抬高型心梗"
→ 编码：I21.401

临床写："非ST抬高心肌梗死"
→ 编码：I21.401

临床写："NSTEMI"
→ 编码：I21.401

例：容易混淆的诊断

临床写："既往乙肝"
→ 可能是 B18.1（慢性携带）或 Z86.1（个人史）或 B18.0（活动性肝炎）——必须看化验指标才能确定

1.3 ICD-10 的三卷：什么时候用哪一卷？

很多新手拿到一套三卷本，第一反应是从第一页翻起按顺序读——这其实效率很低。三卷各有各的调用时机：

❌ 错误做法

从第一卷类目表第一页开始翻，按章节顺序找疾病 → 大海捞针，效率极低

✅ 正确做法

从第三卷索引入手（按字母/拼音找主导词）→ 得到编码 → 回到第一卷核对注释 → 确认无误

卷次名称何时使用核心作用
第一卷类目表索引给出编码后必须回头看核对"包括""不包括"注释；确认编码在分类体系中是否真的匹配
第二卷指导手册遇到编码规则疑问时查阅定义"何时用合并编码""何时用星剑号""死因编码怎么做"等大原则
第三卷字母顺序索引编码的起点——每次都从这里开始按主导词→修饰词的层级结构，引导编码员找到目标类目/亚目

1.4 编码的层级结构——为什么一定要编到最细

以 I21.401 为例：

I            ← 第IX章"循环系统疾病"（字母表示章节归属）
I21          ← 三字符类目"急性心肌梗死"（类目层级，是 DRG 入组的最低要求）
I21.4        ← 四字符亚目"急性非ST段抬高型心肌梗死"（ICD-10 原版的最细层级）
I21.401      ← 国临版扩展码（中国本土化扩展，比原版更细）
⚠️ 关键区别：DRG/DIP 入组依赖的是类目（三字符）级别的编码，但病案首页填报必须到国临版扩展码（通常是五字符或六字符）。编浅了，统计不准、质控扣分；编错了，直接入错组、亏钱。

🔍 三、编码三部曲——深入拆解每一步

1

确定主导词（Leader Word）

从临床诊断中提取"本质词"——即疾病本身是什么。修饰词（急性/慢性/左/右/化脓性）附在后面。

常见主导词类型：

疾病本质阑尾炎、肺炎、糖尿病、高血压、癫痫、贫血
病理改变出血、梗死、骨折、脱位、溃疡、坏死、穿孔
操作/手术切除术、修补术、吻合术、引流术、穿刺（用于手术编码）
2

查第三卷字母顺序索引

按"主导词 → 修饰词（一级修饰→二级修饰→三级修饰）"的层级，逐级缩窄范围。

主导词：阑尾炎
  └ 急性
      └ 化脓性 → 索引指向 K35.8

注意第三卷是独立的一本书，与第一卷类目表分开使用。查索引这一步不需要翻第一卷。

3

返回第一卷核对（最关键一步）

索引给出的编码只是"候选"，必须回到第一卷类目表逐级核对：

  • "包括"（Includes）：确认该码确实涵盖当前诊断
  • "不包括"（Excludes）：是否有看起来相似但应归入其他码的情况
  • 章/节注释：特殊编码规则（如产科章、肿瘤章的特殊说明）

3.1 "不包括"注释——最容易踩的坑

很多编码员跳过"不包括"直接填码，这是导致编码错误的头号原因。"不包括"有两种：

类型含义示例
Excludes1 互斥——两者不能同时使用 I46.1（心脏性猝死）"不包括"心源性猝死（I21-I22）→ 如果患者死于急性心梗发作，应编 I21.x 而非 I46.1
Excludes2 并列——可以同时存在但分类不同 某诊断与另一诊断"不包括"不意味着不能共存，只意味着应分别编码

3.2 星剑号系统（†/*）——双重分类

当一个疾病既有根本病因又有突出临床表现时，ICD-10 使用星剑号双重分类：

  • 剑号（†）：表示病因编码，是主要编码，用于死因统计
  • 星号（*）：表示临床表现编码，是附加编码，用于发病率统计
例：阿尔茨海默病性痴呆
G30.0†  早发性阿尔茨海默病（病因，神经系统）
F00.0*  早发性阿尔茨海默病性痴呆（临床表现，精神科）
⚠️ 实际工作中：剑号为主码，星号为附加码。病案首页二者都填——但主诊断位置放剑号。DRG/DIP 入组以剑号驱动。新手常犯错误：只编了 F00.0* 而漏掉 G30.0†，导致入组偏差。

🧹 四、术语清洗——从"临床话"到"编码话"

术语清洗就是把临床医生口语化、简写的诊断名称，转换成 ICD-10 认的标准诊断名称，再据此编码。这是编码的第一道工序。

临床写法清洗后的标准诊断编码方向清洗要点
"心梗"需区分 STEMI / NSTEMI / 陈旧性I21.x / I25.2简写无法确定具体类型，必须看病程记录
"脑出血"需明确是脑实质出血 / 蛛网膜下腔 / 硬膜下I61.x / I60.x / I62.x必须核对影像报告定位
"既往乙肝"慢性携带者 / 个人史 / 活动性肝炎B18.1 / Z86.1 / B18.1必须看 HBsAg、HBV-DNA、ALT
"头晕"入院时的症状描述R42如果住院期间明确了病因，必须用病因码替代症状码
"脑蛋白变性"可能是脑白质病变 / 阿尔茨海默 / 血管性脑白质病多种可能"脑蛋白变性"不是 ICD-10 标准名称，必须查影像+临床

基础题 · 主导词选择 PRACTICE

请为以下每个临床诊断选出正确的主导词：

  1. 急性化脓性阑尾炎
  2. 左股骨颈骨折
  3. 2型糖尿病伴周围循环并发症
  4. 肺上叶原发性支气管肺癌

✅ 答案：

1. 主导词：阑尾炎——核心疾病是阑尾炎，"急性""化脓性"是修饰词

4. 主导词：肿瘤→恶性→肺——肿瘤查法是"肿瘤→性质（恶性/良性/原位）→解剖部位"

2
基础题 · 术语清洗 PRACTICE

某患者因"头晕"入院，住院期间行头颅 MRI 提示"腔隙性脑梗死"。出院诊断医生写"头晕"。

问题：编码员应该怎么处理？主诊断编什么？

点击查看答案

✅ 规则：入院时使用症状码，但住院期间明确了器质性病因的 → 以确诊替代症状。

✅ 术语清洗：头晕（R42）→ 腔隙性脑梗死（I63.801）

✅ 主诊断：I63.801 腔隙性脑梗死——这是 ICD-10 的基本原则，也是质控检查的高频扣分点。

3
进阶题 · "不包括"规则应用 CHALLENGE

出院诊断"心源性猝死"，病程记录显示患者入院时已无生命体征，心电图提示广泛前壁心梗。

问题：主诊断应编 I46.1（心脏性猝死）还是 I21.x（急性心梗）？为什么？

点击查看答案

✅ 核对卷一：I46.1 "心脏性猝死"类目下有明确的"不包括"注释——"心源性猝死（I21.-、I22.-、I24.-）"。

这意味着：如果心源性猝死是由急性心梗（I21/I22）直接引起的，不应编 I46.1，而应编心梗本身的编码。

✅ 最终编码：I21.0（急性广泛前壁心肌梗死）

💡 核心教训：跳过"不包括"注释就直接编 I46.1，是常见错误。每次核对卷一，第一眼看的就是"不包括"。

进阶题 · 星剑号双重分类 CHALLENGE

出院诊断为"阿尔茨海默病性痴呆"。病程记录显示患者为 65 岁，发病 2 年。

问题：应该编几个码？分别是什么？哪个是主码？

点击查看答案

✅ 需要编两个码：

① G30.0† 早发性阿尔茨海默病（剑号，病因码，主诊断）

② F00.0* 早发性阿尔茨海默病性痴呆（星号，临床表现，附加诊断）

✅ 主诊断是 G30.0†，因为 DRG 入组以剑号驱动。两个码都必须填在病案首页。

5
进阶题 · 完整编码推导 CHALLENGE

某患者出院，临床诊断："右侧基底节区脑出血"。请按完整的编码三部曲给出最终编码，并说明每一步做了什么。

点击查看答案

✅ 第一步 · 主导词：出血 → 然后定位到"脑/颅内"

✅ 第二步 · 查索引：出血 → 脑 → I61.x → 基底节 → I61.0

✅ 第三步 · 核对卷一：I61.0 脑叶出血（包括"皮质出血""脑叶出血"）。基底节属于深部脑结构，应根据影像进一步细分至具体核团——壳核 I61.001 / 丘脑 I61.101。

✅ 最终编码：I61.001 壳核出血 或 I61.101 丘脑出血（依 CT/MR 报告为准）

基础题 · 症状码→确诊码转换 PRACTICE

判断以下场景中主诊断应该用什么，并说明理由：

场景入院诊断住院期间发现你的选择
A腹痛待查胃镜+病理确诊胃窦部腺癌？
B发热血培养确诊肺炎链球菌肺炎？

💡 六、重难点解析：合并编码——什么时候"合二为一"

在 ICD-10 中，两个看似独立的诊断在特定条件下应该用一个合并编码来表达——这就是合并编码（Combination Code）。你会在后续学习（特别是 I 章、J 章）反复遇到这个规则。

合并编码的三个判断条件：

条件说明示例
① ICD-10 明确规定了合并编码 不是编码员自己"觉得有联系"就能合并——必须在第一卷找到了对应的合并条目 II 型糖尿病伴肾病 → E11.2† + N08.3*（有明确合并编码）
② 两个疾病在临床上存在直接的因果关系 A 导致 B，且 ICD-10 提供了这种因果关系的合并编码 高血压性心脏病 → I11.x（高血压+心脏病的合并编码）
③ 必须核对卷一确认 合并编码最终要在第一卷的"包括"注释中得到确认 I11.x 的"包括"明确写明了"高血压性心脏病"涵盖的范围
⚠️ 常见误区：不要把"疾病 A + 疾病 B"随便合并。比如一个高血压患者同时有冠心病，不等于可以用高血压性心脏病（I11.x）的编码——I11.x 是高血压导致的心脏病，而同时存在但无因果关系的两个疾病应该分别编码。

术语清洗：把临床诊断翻译成标准诊断，是编码第一道工序
确诊后必须用病因码替代症状码——除非始终未明确病因
星剑号（†/*）：病因码 + 表现码，两个都要填，剑号为主
至少编到四字符亚目；国临版要编到最细扩展码
合并编码的条件：ICD-10有明确规定 + 有因果关系 + 核对确认
