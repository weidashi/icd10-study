🔑 一、主导词选择——编码的第一道决策

1.1 什么是主导词？为什么它是编码的起点？

主导词（Leader Word）是从临床诊断中提取出来的核心词——它代表疾病本身的本质属性，是进入第三卷索引（字母顺序索引）的"入口钥匙"。你打开第三卷，并不是按拼音翻到"急"字就能找到"急性心肌梗死"——你必须先确定主导词是"梗死"还是"心肌"还是"冠状动脉"，然后按主导词的字母顺序去索引中找。

为什么主导词这么重要？因为第三卷索引是按主导词排列的，同一主导词下汇聚了所有相关的亚目分支。选错主导词 = 进了错误的索引分支 = 找不到正确的编码路径。更糟的是，有时选错主导词也能偶然得到一个编码，但完全经不起卷一的核对——这种"偶然正确"是编码质量的隐形杀手。

核心原则：主导词 = 疾病/损伤/症状本身的本质，而不是它的修饰词、部位、病因或并发症。
修饰词（急性/慢性/左/右/化脓性/复发性/继发性）在索引中是跟着主导词走的，作为修饰层级进一步缩窄编码范围。

1.3 主导词选择的六大实战规则

1

优先用疾病名称，没有则用病理改变

"糖尿病"是疾病名 → 主导词是"糖尿病"；"脑出血"没有疾病名 → 主导词是"出血"。原则：疾病名称优先于病理描述。

2

多病变诊断，取"最本质"的那个

"高血压性心脏病合并心力衰竭"——三个病变（高血压+心脏病+心衰），首选合并编码路径：主导词"高血压"→"伴心脏病"→"伴有心力衰竭"。

3

肿瘤类疾病走专用路径

恶性肿瘤/良性肿瘤/原位癌/动态未定肿瘤 → 主导词统一用"肿瘤"，然后按"性质→部位"的分支检索。不要用"癌""肉瘤"做主导词——索引中有专门的山形括号标注（癌见肿瘤，恶性）。

4

损伤按"损伤"主导词，外因按"意外事故/暴露"

S-T 章损伤编码：主导词"损伤"→部位→类型（开放/闭合）。V-Y 章外因：主导词"意外事故"或"暴露于"。

5

产科章（O00-O99）特殊规则

妊娠、分娩、产褥期并发症的主导词常在"妊娠""分娩""产褥期"下查找，同时注意"并发于妊娠的…"和"为…影响妊娠"的截然不同入口。

6

找不到？换个角度再试

同一种病在索引中可能有多个入口（如"心肌梗死"在"梗死→心肌"和"心肌梗死"都可查到）。如果第一个主导词查不到，试另一个——这是经验积累的过程。

⚠️ 新手最常见的错误：把部位当主导词。
"左下肢静脉曲张"→ 有人会去查"下肢"或"左"，但正确的做法是主导词选"静脉曲张"，然后在修饰层级中找"下肢→左"。索引中"左下肢"没有独立条目——部位是跟着疾病走的。

📚 二、第三卷索引——从主导词到编码的路径

2.1 索引的结构：一棵倒置的"决策树"

第三卷字母顺序索引的编排不是一本书，而是一棵决策树。主导词是树的根节点，一级修饰词是主干分支，二级修饰词是细分支，最终指向的编码是树叶。理解这棵树的结构，比记忆具体编码重要一百倍。

主导词：阑尾炎（Appendicitis） ├─ 急性（acute） │ ├─ 伴有（with） │ │ ├─ 穿孔/破裂（perforation/rupture）→ K35.0 │ │ └─ 腹膜炎（peritonitis）→ K35.0 │ ├─ 化脓性（suppurative）→ K35.8 │ └─ 未特指（unspecified）→ K35.8 └─ 慢性（chronic）→ K36 主导词：肿瘤（Neoplasm） └─ 恶性（malignant）→ 见 肿瘤，恶性 └─ 胃（stomach） ├─ 贲门（cardia）→ C16.0 ├─ 幽门（pylorus）→ C16.4 ├─ 胃窦（antrum）→ C16.3 └─ 未特指（unspecified）→ C16.9

2.2 索引中的关键符号——读懂索引的"密码"

符号/关键字含义使用场景
见（see） 强制跳转——当前条目下没有编码，必须去另一个主导词查 "癌—见 肿瘤，恶性"→告诉你别在"癌"下面找了，去"肿瘤"下面继续
另见（see also） 建议跳转——可能相关但分类不同的其他路径 "胃炎—另见 溃疡，胃"→可能有重叠但分类体系不同，提醒参考
NEC 未在他处分类（Not Elsewhere Classified）——当没有更特异的编码时使用 索引中出现"NEC"时，意味着该编码是"兜底"选择——前提是更特异的编码都不匹配
NOS 未特指（Not Otherwise Specified）——诊断信息不够详细时使用 临床只写了"阑尾炎"，没有"急性/慢性/化脓"等细节 → 用 NOS 编码
† / * 星剑号标记——双重分类提示 剑号（†）= 病因编码（主码），星号（*）= 临床表现编码（附加码）
山形括号 ⟨ ⟩ 表示该主导词在索引中不存在直接条目，需转换 ⟨癌⟩ → 见 肿瘤，恶性

2.3 索引查找的"层级下钻"策略

查索引不是读小说——不需要逐字逐句从头到尾看。推荐的查找策略是层级下钻（Drill-down）：

A

定位主导词——按拼音/字母找到起始页

第三卷按拼音字母顺序排列（中译本）。"阑尾炎"翻到 L 字头，"骨折"翻到 G 字头。找到主导词条目所在的页面。

B

逐级缩窄——沿修饰层级往下走

主导词下面通常有数十个甚至上百个子条目。不需要全看——只看与当前诊断匹配的修饰词。急性/慢性、化脓性/非化脓性、伴/不伴穿孔……每一层修饰都缩小了编码范围。

C

到达编码——看到类目或亚目编号即停

当索引指向一个具体的类目码（如 K35）或亚目码（如 K35.0）时，索引查找完成。不要修改索引给的编码——到卷一核对时再确认是否匹配。

D

走不通回退——换主导词重试

如果当前主导词的修饰层级中找不到匹配的诊断描述，退回第一步换主导词。比如"糖尿病性肾病"在"肾病"下找不到→改查"糖尿病"→伴"肾病"。

例：索引查找 — 急性心肌梗死

主导词：梗死 → 心肌 → 急性

索引层级：

梗死（Infarction）
→ 心肌（myocardium）
→ 急性（acute）→ 见 梗死，心肌 → I21.9

但注意：I21.9 是"未特指部位"——实际编码需根据部位细分至 I21.0-I21.4

例：索引查找 — 2型糖尿病性肾病

主导词：糖尿病 → 2型 → 伴有 → 肾病

索引路径：

糖尿病（Diabetes）
→ 2型（type 2）
→ 伴有（with）
→ 肾病（nephropathy）→ E11.2†

附：还需加 N08.3* 作为星号编码

✔️ 三、核对卷一——编码的最后防线

3.1 为什么索引给的码不能直接填？

第三卷索引是导航工具——它的设计目标是把编码员快速引导到可能的编码位置，而不是给出最终答案。第一卷类目表才是判定标准——包含每个类目/亚目的完整定义、包含范围、排除条目和特殊编码规则。跳过卷一核对直接填索引码，等于只看导航不走路。

专业的说法：索引是"快速定位"，卷一是"最终背书"。非专业术语叫"导航导到了，到地方看清楚再确认"。

3.2 核对卷一的五个必查项

序号必查项要确认什么如果不查会怎样
① 类目标题 该类目是否真的涵盖了当前诊断的疾病范畴 类目标题对不上 → 可能走错了索引分支，整个编码需要推翻重来
② "包括"注释 当前诊断是否在"包括"的列表中——如果有列表，它必须在其中 诊断不在"包括"中 → 该码可能不匹配，需要找正确的类目
③ "不包括"注释 当前诊断是否被"不包括"排除——如果有排除，必须用指明的替代编码 被排除但还用 → 编码错误，质控扣分，DRG入组偏差
④ 章/节/类目注释 特定章（如肿瘤章、产科章、损伤章）可能有特殊编码规则 肿瘤章不区分良恶性 → 漏掉形态学编码；产科章不区分产前产后 → 编码完全错误
⑤ 细目层级 亚目和扩展码是否有更精细的分类（如部位侧别、病原体、分期等） 只编到三字符类目 → 统计粗糙，且国临版要求至少到亚目+扩展码

3.3 "不包括"的两层深度——最容易踩的陷阱

"不包括"（Excludes）是卷一核对中最关键也最容易被跳过的检查。ICD-10 中"不包括"有两种不同含义：

排除类型WHO 术语含义编码行为典型案例
互斥排除 Excludes1 两个诊断不能同时编码——本质上是同一病因的不同表现，或一个诊断是另一个的子集 只能选其中一个编码 I46.1（心脏性猝死）Excludes1: 心源性猝死（I21-I22）→ 若是心梗导致猝死，编I21.x而非I46.1
并列排除 Excludes2 两个诊断分类不同但可以同时存在——只是分类上不放在同一类目下 可以分别编码 某呼吸系统疾病 Excludes2: 某循环系统疾病 → 两者可以共存，分别编码

❌ 不核对卷一的典型错误

诊断"心源性猝死"，索引用"猝死"查到 I46.1，直接填码。结果质控检查发现：病程记录明确记载是心梗导致的猝死 → I46.1 的 Excludes1 指出应编 I21.x。编码错误。

✅ 正确做法

索引引导到 I46.1 → 回卷一核对 → 看到"不包括：心源性猝死（I21.-、I22.-、I24.-）"→ 阅读病程确认是心梗 → 重新以"梗死→心肌→急性"查索引 → 回到 I21.x 核对确认无误。

🔗 四、合并编码规则——何时把两个诊断"合二为一"

4.1 合并编码的本质

合并编码（Combination Code）是 ICD-10 中的一种特殊编码机制：当两个或多个疾病之间存在明确的因果关系或ICD-10 明确规定的组合逻辑时，用一个合并编码替代分别编码。这不是编码员的"个人判断"，而是 ICD-10 分类体系的官方规则。

用一个通俗的类比：如果你看到"高血压引起的心力衰竭"，不能分别编 I10（高血压）和 I50.0（心衰）。ICD-10 提供了一个合并编码 I11.0（高血压性心脏病伴有心力衰竭），它同时表达了"高血压"和"心力衰竭"以及两者间的因果关系。分别编码不仅数据丢失了因果关系，还不符合 ICD-10 的分类规则。

4.2 合并编码的三个判断条件（缺一不可）

条件说明反例（不符合条件）
① ICD-10 第一卷中存在对应的合并编码 合并编码不是编码员"觉得应该合并"就能创建——必须在第一卷类目表中找到了明确的合并条目，通常在"包括"或类目注释中体现 患者有高血压+糖尿病，不能自己编一个"高血压糖尿病综合征"——ICD-10没有这个合并码，应分别编 I10 + E11.9
② 两个疾病之间存在直接的因果关系 A 导致了 B，且临床记录中有明确证据。合并编码表达的是这种因果关系——"由于A导致的B" 患者有高血压，同时因冠心病住院——如果没有证据证明冠心病是高血压引起的，不能编 I11.x（高血压性心脏病）
③ 核对卷一确认——"包括"注释必须匹配 第一卷的"包括"注释必须明确列出当前诊断属于该编码的范畴 I11.x 的"包括"只涵盖"高血压性心脏病"，不涵盖"高血压+心律失常"——后者需分别编码
⚠️ 临床诊断中需合并及省略编码的案例警示：
真实案例：患者因"腹内疝伴肠梗阻"入院。若编码员拆成两个编码（K46.9+K56.7），DRG 入组 GZ13，权重 0.55；若按规则合并为 K46.0（腹内疝伴肠梗阻），入组 GV15，权重 0.39。同一患者、同一次住院，编码合并与否，DRG 支付差额超过 3000 元。拆着编 → 权重虚高 → 医保判定违规；合着编 → 分组准确 → 数据真实。

💡 五、重难点深度解析：NEC 与 NOS —— 编码精度的分水岭

在索引查找和卷一核对的过程中，NEC（未在他处分类）和NOS（未特指）是两个经常出现但极易混淆的概念。它们的正确使用直接影响编码的精度和 DRG 入组准确性。

5.1 概念辨析

维度NEC（Not Elsewhere Classified）NOS（Not Otherwise Specified）
中文含义 未在他处分类——别处没有更合适的编码了 未特指——诊断信息不够详细
本质 这是 ICD-10 分类体系的设计决策——编码员无法改变 这是临床诊断信息不足——但编码员应该尝试获取更多信息
触发条件 当一个疾病确实属于该类目，但 ICD-10 没有为它设立更细的亚目时 临床诊断缺乏关键信息（如"阑尾炎"没有注明急性/慢性/化脓等）
编码员的责任 确认确实没有更特异的编码可用——可以接受 应该尝试联系临床医生补充诊断细节——尽量不用
典型场景 "其他特指的…"类目 —— ICD-10 故意留的"兜底"类目 "未特指的…"类目 —— 诊断细节缺失时的无奈选择

5.2 实战辨析案例

NEC 示例 — 编码员可控范围之外

诊断：嗜酸细胞性胃肠炎

索引路径：胃肠炎 → 嗜酸细胞性 → 指向 K52.8（其他特指的非感染性胃肠炎和结肠炎）

核对卷一：K52.8 是"其他特指"亚目，ICD-10 未设 K52.0-K52.7 之外的更细分类。这是 NEC 的典型场景——编码员无法改变，只能编 K52.8。

NOS 示例 — 编码员应该追问的

诊断：肺炎

索引路径：肺炎 → 未特指 → J18.9

问题：临床没写病原体、没写具体部位、没写临床表现。但病程中可能有痰培养结果、CT 报告等。J18.9 是"未特指的肺炎"——编码员应该主动查看辅助检查报告，争取编到 J12-J18 中更具体的亚目。

一句话总结：NEC = 编码员算了，ICD-10 没给更细的；NOS = 编码员别算了，去问临床要更多信息。
记忆口诀：NEC 是规则认了，NOS 是信息缺了。

✏️ 六、今日案例练习（共 6 题）

前3题为基础题，后3题为进阶题。点击答案查看推理过程。

1
基础题 · 主导词选择 PRACTICE

请为以下每个临床诊断选出正确的主导词，并说明选择的理由：

  1. 急性化脓性胆管炎
  2. 右肺上叶周围型腺癌
  3. 2型糖尿病伴周围神经病变
  4. 左肱骨干闭合性骨折
  5. 妊娠合并子痫前期（重度）
点击查看答案

✅ 1. 主导词：胆管炎——"急性""化脓性"是修饰词。推理：疾病本质是"炎"（胆管发炎），修饰词描述发病速度和性质。

✅ 2. 主导词：肿瘤→恶性→肺——肺癌走肿瘤索引路径，不是"癌"也不是"肺"。推理：ICD-10 肿瘤索引用"肿瘤"做主导词（癌见肿瘤，恶性）。

✅ 3. 主导词：糖尿病——周围神经病变是糖尿病的并发症。推理：合并编码以病因（糖尿病）为主导词，去查伴发症。

✅ 4. 主导词：骨折——病理改变是骨折，"左""肱骨""干""闭合"全是修饰词。推理：损伤以病理改变为入口。

✅ 5. 主导词：妊娠——产科章（O章）特殊规则，妊娠并发症以"妊娠"为主入口。推理：O章按"妊娠→并发→子痫前期"路径检索。

2
基础题 · 索引查找完整路径 PRACTICE

请写出以下三个诊断的完整索引查找路径（主导词 → 一级修饰 → 二级修饰 → 索引给出的编码）：

  1. 急性ST段抬高型下壁心肌梗死
  2. 慢性阻塞性肺疾病伴急性下呼吸道感染
  3. 甲状腺乳头状癌
点击查看答案

✅ 1. 急性ST段抬高型下壁心肌梗死
主导词：梗死 → 心肌 → 急性 → 透壁性 → 下壁 → I21.1
推理：I21.1 是急性透壁性下壁心肌梗死——"透壁性"即ST段抬高型。注意还需核对国临版扩展码细分。

✅ 2. COPD伴急性下呼吸道感染
主导词：病 → 肺 → 阻塞性 → 慢性 → 伴有 → 急性 → 下呼吸道感染 → J44.0
推理：J44.0 明确=慢性阻塞性肺病伴有急性下呼吸道感染，是合并编码的经典应用。

✅ 3. 甲状腺乳头状癌
主导词：肿瘤 → 恶性 → 甲状腺 → C73
推理：部位码 C73。形态学码需另加：M8050/3（乳头状癌）。形态学编码来自肿瘤形态学索引，与部位编码分开查找。

3
基础题 · 术语清洗+编码规则判断 PRACTICE

某患者出院诊断写"头晕、高血压"。病程显示：患者因"头晕3天"入院，住院期间测血压 185/110mmHg，心脏超声提示左心室肥厚，临床诊断为"高血压性心脏病（NYHA II级），头晕由血压控制不佳所致"。

问题：

  1. 临床写的"头晕、高血压"需要做术语清洗吗？如果需要，应该怎么清洗？
  2. 最终应该编哪几个编码？哪个是主诊断？
点击查看答案

✅ 1. 术语清洗：必须清洗！"头晕"已被证实由高血压所致 → 不能用症状码 R42。"高血压"需要根据心脏超声结果进一步明确：应清洗为"高血压性心脏病（NYHA II级）"。

✅ 2. 最终编码：

主诊断：I11.0 高血压性心脏病伴有心力衰竭（NYHA II级提示轻度心衰）
——如果有 I11.0 的"包括"确认涵盖 NYHA II 级，则不需要另编 I50。

核心规则：确诊后必须用病因码替代症状码；高血压+心脏病的因果关系需合并编码。

4
进阶题 · "不包括"规则陷阱 CHALLENGE

某患者因"急性广泛前壁心肌梗死"入院，住院第3天发生"心室颤动、心脏骤停"，经心肺复苏后恢复自主心律。出院时医生写了两个诊断："急性广泛前壁心梗"和"心脏骤停成功复苏"。

问题：

  1. 索引查"心脏骤停"会指向哪个编码？
  2. 核对卷一后，这个编码还能不能用？为什么？
  3. 正确的编码方案是什么？
点击查看答案

✅ 1. 索引查"心脏骤停"（Cardiac arrest）→ 指向 I46.9（心脏停搏，未特指）或 I46.0（心脏停搏成功复苏）。

✅ 2. 核对卷一：I46 类目的 Excludes1 明确指出——"心源性休克（R57.0）"、"并发于……的心脏停搏"，特别排除了由急性心梗引起的心脏骤停。也就是说如果心梗是本次住院的主要病因，心脏骤停作为心梗的并发症不应单独编码 I46。

✅ 3. 正确方案：主诊断 I21.0（急性广泛前壁心肌梗死）——心梗才是根本病因。心脏骤停作为心梗的并发症，在 I21.x 的"包括"中已被涵盖（急性心梗的并发症不需要额外编码）。

教训：索引给 I46.x → 卷一 Excludes1 排除 → 回到心梗做唯一的诊断。跳过"不包括"就编 I46.x + I21.0 两个码，是典型的多编错误，会导致 DRG 入组产生歧义。

5
进阶题 · 合并编码 vs 分别编码 CHALLENGE

分析以下三个场景，判断应该合并编码还是分别编码，并说明理由和最终编码方案：

场景临床诊断附加信息
A胃溃疡、上消化道出血胃镜明确：胃窦部溃疡（Forrest IIa），活动性出血，溃疡基底可见血管残端
B高血压、冠心病造影显示：前降支中段狭窄85%，高血压病史10年。无证据表明冠心病由高血压直接引起
C2型糖尿病、慢性肾功能不全IV期肾穿刺活检：糖尿病肾病——结节性肾小球硬化。空腹血糖 11.2mmol/L，HbA1c 9.1%
点击查看答案

✅ A：应该合并编码。胃溃疡 Forrest IIa是活动性出血溃疡 → 出血是溃疡的直接并发症 → K25.4（胃溃疡伴出血）是合并编码。分别编 K25.9+K92.2 是不正确的，因为 K25.4 的"包括"已明确涵盖"胃溃疡伴出血"。

✅ B：应该分别编码。虽然患者同时有高血压和冠心病，但没有证据证明冠心病是高血压引起的（前降支狭窄是动脉粥样硬化所致，而非高血压性心脏病）。I11.x（高血压性心脏病）的"包括"不涵盖"高血压+冠心病"——应分别编 I10 + I25.1。

✅ C：应该合并编码（星剑号）。肾穿刺明确是糖尿病肾病 → 因果关系成立 → 主码 E11.2†（2型糖尿病伴肾病）+ 附加码 N08.3*（糖尿病性肾病）。不能分别编 E11.9+N18.4——那样丢失了因果关系，DRG 可能入不同的组。

判定口诀：有直接因果+ICD-10有合并码=合并；无因果或无合并码=分别。

6
进阶题 · 综合编码推导——完整三步法 CHALLENGE

以下是一位患者的完整出院诊断，请按编码三部曲给出所有诊断的 ICD-10 编码（包括主诊断和其他诊断），并说明每一步的决策过程：

出院诊断：

  1. 急性胆囊炎伴胆囊结石
  2. 2型糖尿病
  3. 高血压病3级（极高危）

病程摘要：患者因右上腹剧痛3天入院。B超提示：胆囊多发结石，胆囊壁增厚，胆囊周围少量积液。急诊行腹腔镜胆囊切除术。住院期间发现血糖升高（空腹8.9），内分泌科会诊诊断2型糖尿病。既往有高血压病史5年，入院时血压165/100mmHg。

点击查看答案

推导过程：

主诊断判断：本次住院的主要原因是胆囊结石伴胆囊炎 → 急性胆囊炎是入院主因 → 应为主诊断。

诊断① 急性胆囊炎伴胆囊结石：
① 主导词：胆囊炎 → 急性 → 伴有 → 结石（或查"结石→胆囊→伴有→胆囊炎"）
② 索引 → K80.0（胆囊结石伴急性胆囊炎）
③ 核对卷一：K80.0 "包括"急性胆囊炎伴胆囊结石 ✔ → 最终：K80.000（国临版扩展）
④ 注意：不能分别编 K81.0 + K80.2——K80.0 是合并编码，已涵盖"结石+急性胆囊炎"

诊断② 2型糖尿病：
① 主导词：糖尿病 → 2型 → 未提及并发症 → E11.9
② 核对卷一：E11.9 "2型糖尿病不伴并发症" ✔
③ 归为其他诊断

诊断③ 高血压病3级：
① 主导词：高血压 → 原发性 → I10
② 核对卷一："包括"高血压病（原发性）✔，未提及心脏病/肾病
③ 本次会诊未发现靶器官损害证据 → I10 合适
④ 归为其他诊断。注意："极高危"是临床危险分层，不是 ICD-10 编码要素。

最终编码方案：

主诊断：K80.000 胆囊结石伴急性胆囊炎
其他诊断：E11.900 2型糖尿病
其他诊断：I10.x00 原发性高血压

✅ 七、今日重点回顾

主导词选疾病的"本质"——病名优先，病理次之，部位只做修饰
肿瘤走"肿瘤→性质→部位"专用索引路径；癌见肿瘤恶性
索引是导航不是终点——给的是候选编码，必须回卷一核对
核对卷一先看"不包括"——排除最危险，跳过等于闭眼编码
"见"=强制跳转，"另见"=建议参考，NEC≠NOS
合并编码三条件：ICD-10有合码 + 有直接因果 + 卷一确认
查不到换主导词重试——同一种病在索引中可能有多个入口
每一步都要脑中有"为什么"——能说出推导过程才算真掌握
